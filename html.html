HTML — Structure and Semantics

HTML (HyperText Markup Language) provides the structural foundation of web pages. Good HTML is semantic, accessible, and organized. It uses elements that describe meaning (e.g., <header>, <nav>, <main>, <article>).

Core Principles
  • Use headings (<h1>–<h6>) in logical order.
  • Wrap main content in <main> and navigation in <nav>.
  • Prefer semantic containers (<section>, <article>, <aside>, <footer>).
  • Add alt text to images for screen readers.
  • Label form inputs; associate with <label for>.
Example Markup
<article class="post">
  <header>
    <h2>Post Title</h2>
    <p>Published <time datetime="2025-08-25">Aug 25, 2025</time></p>
  </header>
  <p>Clear, semantic HTML improves accessibility and SEO.</p>
  <footer>© Author</footer>
</article>

Forms and Accessibility

<form>
  <label for="email">Email</label>
  <input id="email" type="email" required aria-describedby="help" />
  <div id="help">We'll never share your email.</div>
  <button type="submit">Submit</button>
</form>
