What is Full Stack Development?

Full stack development refers to building both the front end (client‑side) and the back end (server‑side) of a web application, plus the glue that connects them (APIs, databases, deployment). A full stack developer can design data models, implement APIs, craft accessible UIs, and ship the product to production.

Typical Layers
  • UI (Front End): HTML for structure, CSS for presentation, JavaScript for behavior.
  • API layer: REST/GraphQL endpoints that the UI calls.
  • Business logic: Validates input, enforces rules, orchestrates workflows.
  • Data layer: Databases (SQL/NoSQL), ORMs, caching.
  • DevOps & deployment: CI/CD, version control, testing, hosting, monitoring.
Responsibilities
  • Design accessible, responsive interfaces.
  • Model data and write efficient queries.
  • Build secure endpoints and handle authentication/authorization.
  • Write tests, debug across the stack, and profile performance.
  • Deploy, monitor, and iterate quickly with version control.

Example Architecture (Simple To‑Do App)

This ePortfolio includes a working front‑end To‑Do app. In a full stack version, the front end would call a back‑end API to persist tasks, and a database to store them. A basic flow:

Client (JavaScript) → /api/todos (REST endpoints) → Database (tasks table)

Common Tools

  • Front end: HTML5, CSS3, JavaScript (ES6+), Bootstrap, React/Vue (beyond this assignment).
  • Back end: Node.js/Express, Python/Django or Flask, Java/Spring.
  • Databases: PostgreSQL/MySQL (relational), MongoDB (document).
  • DevOps: Git & GitHub, CI/CD, Docker, cloud providers.
